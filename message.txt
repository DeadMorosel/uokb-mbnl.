1 loka: Тип Boolean (Логический). Это как выключатель света — у него есть всего два положения: либо включено (true), либо выключено (false).

2 loka: Тип Number (Числа). Используется для любых чисел — и целых (как 5), и дробных (как 5.5). Если число ну очень огромное, есть редкий отдельный тип BigInt.

3 loka: Тип String (Строки). Это любой текст. Чтобы компьютер понял, что это текст, его просто заматывают в кавычки: 'привет', "мир" или `тест`.

4 loka: Тип Object (Объекты). В JavaScript у массивов нет своего личного типа. Массив — это просто особая разновидность объекта, которая умеет хранить списки вещей по порядку.

5 loka: Тип Object (Объекты). Даты — это тоже не какой-то особый тип данных, а встроенный в язык объект, который умеет считать время и дни.

6 loka: Ключевое слово const. Это способ создать «железную» коробку. Положил туда значение один раз — и поменять его внутри кода уже не получится.

7 loka: Создание числовой переменной. Пишем let, имя переменной и через равно даем ей число. Компьютер сам поймет, что это число:
let myNumber = 42;

8 loka: Глобальная область. Если объявить переменную через let или const на самой первой строчке файла (вне любых функций), её будет видно вообще везде в этом файле.

9 loka: Блочная область (внутри {}). Если создать переменную через let или const внутри фигурных скобок (например, внутри цикла или условия), то за пределами этих скобок она сотрется и работать не будет.

10 loka: Константа (const). Защищает переменную от перезаписи. Если попытаться насильно записать туда новое значение, код выдаст ошибку и сломается.

11 loka: Создание функции. Функция — это запечатанная инструкция (команда). Пишем слово function, даем ей имя и ставим скобки:
function привет() {}

12 loka: Вызов функции. Чтобы запустить отложенную инструкцию, пишем её имя и круглые скобки:
myFunction();

13 loka: Стрелочная функция. Это просто современный, укороченный способ записать функцию, используя стрелочку =>:
const мояФункция = () => {};

14 loka: Анонимная функция. Это функция-безымяшка. У неё нет своего имени, её обычно сразу дарят другой функции или кладут в переменную:
function() {}

15 loka: Передача функции в аргумент. В JavaScript функции можно передавать как обычные вещи (числа или строки) внутрь других функций, чтобы те запустили их позже.

16 loka: Создание массива. Проще всего открыть квадратные скобки [] и перечислить внутри всё через запятую:
const мойМассив = [1, 2, 3];

17 loka: Первый элемент массива. Компьютер считает с нуля. Поэтому у самого первого элемента индекс всегда 0:
myArray[0]

18 loka: Метод push(). Эта команда берет новую вещь и запрыгивает с ней в самый хвост (конец) вашего списка-массива.

19 loka: Метод pop(). Эта команда, наоборот, отрезает и выбрасывает самый последний элемент из конца массива.

20 loka: Создание пустого объекта. Объект — это шкафчик, где хранятся вещи в формате «имя: значение». Пустой шкаф создается фигурными скобками {}:
const мойОбъект = {};

21 loka: Знак равенства =. В программировании один знак = не сравнивает, а берет значение справа и записывает в переменную слева.

22 loka: Строгое равенство ===. Три равно проверяют честно: равны ли значения и совпадает ли их тип. Строка "5" и число 5 для него будут разными вещами.

23 loka: Строгое неравенство !==. Проверяет, что две вещи абсолютно разные по значению или по типу.

24 loka: Плюс + для чисел. Обычная математика: складывает два числа между собой.

25 loka: Плюс + для текста. Если склеить текст с текстом (или текст с числом), они просто скрепятся в одну длинную строчку: "супер" + "код" станет "суперкод".

26 loka: Объявление строки. Просто пишем любой текст внутри кавычек:
const текст = "Привет";

27 loka: Свойство .length. Пишется в конце строки без круглых скобок и мгновенно говорит, сколько букв (символов) в этой строке.

28 loka: Метод .slice(). Позволяет «откусить» кусок от строки, указав, с какой по какую букву тебе нужно забрать.

29 loka: Метод .toUpperCase(). Делает все буквы в строке КАПСОМ (большими).

30 loka: Метод .toLowerCase(). Делает все буквы в строке маленькими.

31 loka: Строку в число. Проще всего поставить обычный плюс перед строкой: +"123" превратится в настоящее число 123.

32 loka: Число в строку. Вызываем у числа команду .toString(), и оно превращается в текст, который теперь нельзя складывать математически.

33 loka: Логическое в строку. Команда .toString() превратит логическое true в обычное слово "true".

34 loka: Оператор typeof. Инструмент-сканер. Пишешь перед переменной typeof x, и он текстом отвечает, что там лежит: "number", "string" и так далее.

35 loka: Превращение объекта в JSON. Команда JSON.stringify(объект) берет сложный объект и переделывает его в одну длинную строку, чтобы её можно было отправить по интернету.

36 loka: Условие if. Говорит компьютеру: «Если то, что в скобках — правда, то сделай код из фигурных скобок»:
if (баланс > 0) { купить() }

37 loka: Переключатель switch. Удобно, когда у тебя есть одна переменная и много четких вариантов для неё. Заменяет кучу скучных if / else.

38 loka: Цикл for. Крутит код по кругу заданное количество раз (например, ровно 10 раз), пока работает встроенный счетчик.

39 loka: Цикл while. Крутит код по кругу до тех пор, пока условие в скобках остается правдой. Может крутиться бесконечно, если условие никогда не испортится.

40 loka: Команда break. Кнопка экстренного выхода. Если компьютер видит внутри цикла слово break, он бросает цикл и бежит дальше.

41 loka: Конструкция try...catch. Подушка безопасности. Код в try пытается выполниться, а если там случится авария, программа не закроется, а управление уйдет в блок catch, где ошибку можно починить или записать.

42 loka: Оператор throw. Ручной вызов аварии. Если в коде пошло что-то не так, мы можем сами закричать «Ошибка!» с помощью throw new Error("текст").

43 loka: Блок finally. Дополнение к блоку аварий. Код внутри finally выполнится на 100% в любом случае: и если всё прошло идеально, и если всё сломалось.

44 loka: Проверка ошибки в catch. Когда ловится ошибка, внутри catch(error) у нас появляется объект этой ошибки. У него можно спросить имя или текст сообщения, чтобы понять, что сломалось.

45 loka: Свой класс ошибки. Создание своей личной ошибки (например, ОшибкаОплаты) на основе стандартной с помощью слова extends:
class ОшибкаОплаты extends Error {}

46 loka: Слово export. Ставится перед функцией, чтобы разрешить другим файлам брать её и использовать у себя.

47 loka: Слово import. Притаскивает в текущий файл функцию, которую кто-то разрешил взять из другого файла с помощью export.

48 loka: Импорт всего (* as). Забирает из чужого файла вообще всё, что там разрешено к экспорту, и аккуратно складывает в один общий объект.

49 loka: Главный экспорт (export default). Говорит, что эта функция — самая главная в файле. Такой экспорт может быть только один на весь файл.

50 loka: Импорт главного экспорта. Когда мы забираем главную вещь из файла, нам не нужны фигурные скобки, и мы можем назвать её у себя в файле абсолютно любым именем.

51 loka: Метод document.getElementById(). Находит на странице один конкретный элемент, у которого в HTML прописан уникальный id:
const block = document.getElementById('myId');

52 loka: Метод document.getElementsByClassName() или современный document.querySelectorAll(). Собирают со страницы целую коллекцию (список) элементов, у которых совпадает имя класса.

53 loka: Свойство textContent. Позволяет стереть старый текст внутри элемента и записать туда новый:
element.textContent = "Новый текст";

54 loka: Команда element.classList.add(). Навешивает на элемент новый CSS-класс, чтобы применить к нему новые стили или анимацию.

55 loka: Команда element.classList.remove(). Стирает CSS-класс с элемента, убирая привязанные к этому классу стили.

56 loka: Метод addEventListener(). Подслушивает действия пользователя. Мы говорим ему, какое событие ждать (например, клик мышкой 'click'), и какую функцию запустить, когда это произойдет:
element.addEventListener('click', myFunction);

57 loka: Метод removeEventListener(). Заставляет элемент перестать слушать событие и реагировать на него. Работает только если передать ему ту же самую функцию, что была добавлена.

58 loka: Метод event.preventDefault(). Говорит браузеру: «Не делай то, что ты делаешь обычно». Например, останавливает перезагрузку страницы при отправке формы или не дает перейти по ссылке при клике.

59 loka: Метод event.stopPropagation(). Останавливает «всплытие» события. Клик на кнопку больше не полетит вверх к её родительским блокам и не активирует их обработчики.

60 loka: Свойство event.type. Показывает строку с названием события, которое прямо сейчас произошло (например, вернет "click" или "keydown").

61 loka: Слово async перед функцией. Магическое слово, которое превращает обычную функцию в асинхронную. Такая функция внутри себя позволяет ставить код «на паузу» и всегда возвращает обещание (промис):
async function load() {}

62 loka: Оператор await. Ставится только внутри async-функции прямо перед долгим процессом (например, скачиванием файла). Он буквально замораживает выполнение кода на этой строчке, пока процесс не завершится.

63 loka: Конструктор new Promise(). Это коробка-обещание. Мы даем ей задачу и две встроенные кнопки: resolve (если всё прошло успешно) и reject (если случилась беда):
const myPromise = new Promise((resolve, reject) => {});

64 loka: Метод .then(). Прицепляется к промису и ждет, пока тот успешно выполнится. Как только обещание сработает, код внутри .then() заберет результат.

65 loka: Метод .catch(). Прицепляется к промису для ловли ошибок. Если в промисе что-то сломалось или сработал reject, управление мгновенно улетит сюда.

66 loka: Регулярное выражение (RegExp). Это специальный шаблон для поиска букв или цифр в тексте. Проще всего создать его, зажав правила между двумя косыми чертами:
const pattern = /[0-9]/;

67 loka: Метод regex.test(). Берет регулярное выражение и проверяет строку. Если в строке нашлось то, что мы искали, выдает true, если нет — false.

68 loka: Метод string.match(). Ищет в строке совпадения по нашему шаблону и возвращает их в виде готового массива.

69 loka: Метод string.replace(). Находит по шаблону кусок текста в строке и заменяет его на другое слово.

70 loka: Метод string.search(). Ищет шаблон в строке и возвращает номер (индекс) первой буквы, где началось совпадение. Если ничего не нашлось, вернет -1.

71 loka: Конструктор new Date(). Создает специальный объект-календарь, с которым можно удобно работать.

72 loka: Пустой new Date(). Если вызвать его без цифр внутри скобок, он автоматически захватит точное текущее время и дату с твоего устройства.

73 loka: Метод .getFullYear(). Вытаскивает из объекта даты чистый четырехзначный год (например, 2026).

74 loka: Метод .getMonth(). Вытаскивает номер месяца. Важно: месяцы считаются с нуля, поэтому январь — это 0, а декабрь — 11.

75 loka: Метод .getDay(). Возвращает номер дня недели. Во многих странах отсчет идет с воскресенья, поэтому воскресенье — это 0, понедельник — 1, а суббота — 6.

76 loka: Метод Math.floor(). Математический пол. Обрубает дробную часть и округляет число строго вниз. Например, 4.9 превратит в 4.

77 loka: Метод Math.round(). Стандартное школьное округление. Если после точки 5 и больше — округлит вверх, если меньше — вниз.

78 loka: Метод Math.ceil(). Математический потолок. Округляет любое число с дробью строго вверх до ближайшего целого. Например, 4.1 превратит в 5.

79 loka: Метод Math.random(). Выдает случайное дробное число от 0 (включительно) до 1 (не включая саму единицу), вроде 0.4152....

80 loka: Оператор  или Math.pow(). Возводит число в нужную степень. Запись 2  3 означает два в кубе (будет 8).

81 loka: Метод Array.isArray(). Единственный надежный способ проверить массив. Обычный typeof скажет, что массив — это объект. А Array.isArray(x) честно ответит true.

82 loka: Проверка через typeof x === 'object'. Проверяет, является ли значение объектом. Но нужно помнить про ловушку: для значения null этот сканер тоже вернет "object", поэтому обычно дописывают проверку: x !== null.

83 loka: Метод concat() или деструктуризация [...arr1, ...arr2]. Склеивает несколько массивов в один общий большой список.

84 loka: Коллекция new Set(). Объект Set умеет хранить только уникальные вещи. Если закинуть туда массив с дубликатами, а потом вернуть обратно в массив, все повторения исчезнут:
const unique = [...new Set(myArray)];

85 loka: Метод sort(). Сортирует элементы массива. По умолчанию он сортирует их как строки (по алфавиту). Чтобы правильно отсортировать числа, внутрь нужно передать функцию-помощник: arr.sort((a, b) => a - b).

86 loka: Функция Number.isNaN(). Специальная проверка для значения NaN (Not a Number / Не число). Обычное сравнение типа x === NaN не работает, потому что в JS NaN не равен даже самому себе.

87 loka: Сравнение x === undefined. Проверяет, является ли переменная абсолютно пустой и неопределенной (когда её создали, но ничего внутрь не положили).

88 loka: Сравнение x === null. Проверяет, занулена ли переменная намеренно (когда разработчик сам написал x = null, чтобы показать, что тут пусто).

89 loka: Проверка typeof x === 'number'. Проверяет, лежит ли в переменной число. Стоит помнить, что NaN технически тоже имеет тип "number".

90 loka: Проверка typeof x === 'string'. Сканирует переменную и подтверждает, что внутри находится обычная текстовая строка.

91 loka: Функция fetch(url). Отправляет стандартный GET-запрос на сервер, чтобы притащить оттуда какие-то данные (например, список товаров).

92 loka: fetch с настройками. Чтобы отправить данные на сервер (POST), в fetch вторым аргументом передают настройки: указывают метод 'POST' и тело запроса body:
fetch(url, { method: 'POST', body: data });

93 loka: Метод .json(). Ответ от fetch нельзя прочитать сразу. Нужно вызвать у ответа метод response.json(), который превратит серверный текст в готовый JS-объект.

94 loka: Проверка response.ok. Сервер может ответить ошибкой (например, 404), но для fetch сам запрос прошел успешно. Поэтому надо вручную проверять свойство response.ok — если там false, значит на сервере случилась беда.

95 loka: Связка async/await с fetch. Самый красивый способ делать запросы без бесконечных .then():
const response = await fetch(url);
const data = await response.json();

96 loka: Объект new FileReader(). Специальный инструмент в браузере, который умеет брать файлы с компьютера пользователя (например, загруженную картинку) и читать то, что у них внутри.

97 loka: Метод readAsDataURL(). Команда для FileReader. Она берет файл и превращает его в одну длинную строчку кода (base64). Эту строчку потом можно легко вставить прямо в тег картинки <img src="...">, чтобы показать её на экране.

98 loka: Событие onload. У FileReader чтение файлов происходит не мгновенно. Нужно повесить событие reader.onload = () => {}, которое сработает ровно тогда, когда файл полностью прочитается. Результат будет лежать в reader.result.

99 loka: Проверка через window.File. Чтобы узнать, умеет ли старый браузер пользователя работать с файлами, проверяют наличие встроенных инструментов: if (window.File && window.FileReader).

100 loka: Конструктор new Blob(). Создает сырой кусок бинарных данных (Blob — Binary Large Object). По сути, это позволяет прямо в коде собрать виртуальный файл (например, текстовый или картинку) из строк или массивов:
const myBlob = new Blob(["Привет, мир!"], { type: 'text/plain' });
101 loka: Функция высшего порядка. Это обычная функция, которая делает одну из двух вещей (или обе): либо принимает другую функцию как аргумент (коробку с командами), либо отдает функцию как результат своей работы.

102 loka: Метод map(). Пробегает по всему массиву, берет каждый элемент, изменяет его по вашему правилу и собирает из результатов абсолютно новый массив такой же длины.

103 loka: Метод reduce(). Накопитель. Он бежит по массиву и сжимает все его элементы в одно единственное значение (например, считает сумму всех чисел или склеивает их в одну строку).

104 loka: Метод every(). Строгий проверяющий. Он смотрит на все элементы массива. Если абсолютно каждый элемент подходит под ваше условие, он выдаст true. Если хотя бы один не подошел — вернет false.

105 loka: Метод Object.assign() или спред-оператор {...obj1, ...obj2}. Копирует все свойства и значения из одного или нескольких объектов в другой, собирая их вместе.

106 loka: Метод Object.keys(). Заглядывает внутрь объекта, вытаскивает оттуда только имена всех свойств (ключей) и складывает их в обычный массив строк.

107 loka: Метод Object.values(). Делает то же самое, но игнорирует имена свойств и собирает в массив только сами значения, которые за этими свойствами были закреплены.

108 loka: Метод Object.entries(). Разбирает объект на пары. Он возвращает большой массив, внутри которого лежат маленькие массивы из двух элементов: [ключ, значение].

109 loka: Оператор in или метод hasOwnProperty(). Позволяет быстро узнать, есть ли у объекта конкретное свойство. Запись 'age' in user выдаст true, если внутри объекта user есть свойство age.
